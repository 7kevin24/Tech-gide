Mkdocs and Material theme

Info

Python和Git的使用已经很广，网上相关教程很丰富，在此不再赘述，接下来的步骤均默认大家已经正确安装Python和Git。

Start

MKdocs真的将制作静态网页的门槛降到了最低，你可以用他整理自己的赛博笔记本，上传Blogs，也可以和三五好友一起维护一个小小wiki，多是一件诱人的事情，还不赶紧来试试？

  1. 安装Mkdocs，在任何终端执行 
    pip install mkdocs

  2. 在你想要保存项目的文件夹内（最好是空文件夹）内打开终端，执行： 

    mkdocs new <your-project-name>
cd <your-project-name>

  3. 此时你可以看见一个docs文件夹用于存放你的各种文件和资源、一个.yml文件用于配置几乎网页的功能。docs文件夹内有一个index.md，这将展示网站首页，你可以稍作修改，然后执行： 

    mkdocs serve

  4. 操作正确的话你已经可以在http://127.0.0.1:8000/打开这个网页了，并且看见index.md经渲染后的页面。
  5. 想要添加其他页只需要将.md文件放进docs文件夹即可，真的非常简单。

网页托管

为了能供他人访问，我们还需要将网页托管。主流的方法有Github page和Cloudflare，以Github page为例：

  1. 在自己的github上新建一个repo，名字任意
  2. 在project所在的文件夹打开控制台并执行，注意替换<your-username>和<your-repo> 
    git init
git add .
git remote add origin https://github.com/<your-username>/<your-repo>.git 
mkdocs gh-deploy
  3. 稍作等待就可以在https://<your-user-name>.github.io/<your-repo>/打开自己的网页了！

Customize

目前，我们MKdocs网页并不好看，我们需要定制属于自己的模板，虽然这个过程比较费时费力，但却是一个一劳永逸的过程。我使用的主题Material，它内置了很多好用的插件，稍做配置即可实现很多有意思的功能，同时，也提供了丰富的可指定选项。总之，非常推荐Material for Mkdocs。其他Mkdocs资源可以参考https://github.com/mkdocs/catalog。

下面给大家推荐一些我觉得有用的配置，旨在帮大家节省一些时间（你甚至可以直接copy我的.yml文件，.css和.js文件拿去用）。但是，我仍然强烈推荐根据需求阅读官方Documentation并善用搜索。因为它页面更好看，内容更全面，操作步骤也都非常详细，并且一直在维护更新。

  • 亮暗主题切换+自定颜色

我的模板

mkdocs.yml（点击展开） 
site_name: Kevin's Tech guide
markdown_extensions:
- pymdownx.superfences
- pymdownx.highlight:
      anchor_linenums: true
      line_spans: __span
      pygments_lang_class: true
- pymdownx.inlinehilite
- pymdownx.emoji:
      emoji_index: !!python/name:material.extensions.emoji.twemoji
      emoji_generator: !!python/name:material.extensions.emoji.to_svg
- attr_list
- md_in_html
# below is for the admonition extension
- admonition
- pymdownx.details
- pymdownx.superfences
- pymdownx.arithmatex:
      generic: true
plugins:
- glightbox
theme:
